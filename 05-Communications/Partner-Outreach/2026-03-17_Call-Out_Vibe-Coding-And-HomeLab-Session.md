## What is this?

We're looking for people with skills, knowledge, and interest in agentic workflows and home lab infrastructure to join a working session this week or next.

The session will cover:

- **Agentic workflow coding** — how we route messages, combine agentic code with visual documentation and specifications, and build visual interfaces to workflows
- **Peer-to-peer workflow coding** — moving beyond Obsidian-only workflows to something more robust
- **HomeLab networking** — connecting 3–4 home labs into a peer-to-peer network for shared processing
- **Low-cost deployment** — making this work across hosted VPS, edge, and home lab environments

## Why now?

We have a concrete problem to solve. Right now, our voice transcript pipeline works like this:

1. Team members send voice messages on Signal, Telegram, WhatsApp
2. Audio files are saved into channel-specific folders on NextCloud
3. MacWhisper transcribes them locally
4. Transcripts are summarised with action points and committed to our planning repo

**The limitation:** this only runs when one person's machine is on. We need this to be distributed — running across home labs so it works regardless of who's online. This is a real, immediate use case for the HomeLab network David has been proposing.

Beyond transcripts, the same pattern applies to all our media processing — video editing, document transformation, and the Fathom recording pipeline.
